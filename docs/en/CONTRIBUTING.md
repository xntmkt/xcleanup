# Contributing

## General principles
- Follow the coding standards and review process.
- Do not commit secrets or sensitive data.

## Contribution workflow
1. Create a branch with the convention: `feat/`, `fix/`, `chore/`.
2. Write tests for new changes.
3. Run: `composer lint`, `composer stan`, `composer test`.
4. Open a Pull Request with a clear description.

## Coding standards
- PHP 8.2, PSR-12.
- Complete docblocks for public APIs.

## Bug reports
- Use the bug report template in `docs/vi/templates/bug_report.md`.
